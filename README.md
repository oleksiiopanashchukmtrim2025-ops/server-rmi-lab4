# 🧪 Лабораторна робота №3

## Java RMI Chat Client

---

## 📌 Опис

Даний проєкт є реалізацією клієнтського застосунку для обміну повідомленнями з використанням технології **Java RMI (Remote Method Invocation)**.

Клієнт підключається до RMI-сервера, отримує віддалений об'єкт та виконує виклики методів для взаємодії з іншими користувачами.

---

## ⚙️ Можливості

Реалізовано такі функції:

* 🔹 Перевірка з'єднання (`ping`)
* 🔹 Відправка тексту на сервер (`echo`)
* 🔹 Авторизація користувача (`login`)
* 🔹 Отримання списку активних користувачів (`list`)
* 🔹 Надсилання повідомлень (`msg`)
* 🔹 Передача файлів (`file`)
* 🔹 Приймання повідомлень та файлів (через Timer)
* 🔹 Завершення роботи клієнта (`exit`)

---

## 🚀 Запуск проєкту

### 1️⃣ Запуск сервера

Перейдіть у папку:

```bash
server.rmi
```

Запустіть файл:

```bash
start.cmd
```

---

### 2️⃣ Запуск клієнта

Відкрийте проєкт у **IntelliJ IDEA** та запустіть:

```bash
Main.java
```

---

## 💻 Доступні команди

```text
ping
echo <текст>
login <логін> <пароль>
list
msg <користувач> <повідомлення>
file <користувач> <шлях_до_файлу>
exit
```

---

## 📂 Збереження файлів

Усі отримані файли автоматично зберігаються у папці:

```text
/received
```

(створюється автоматично під час роботи програми)

---

## ⚠️ Важливо

* Сервер повинен бути запущений перед стартом клієнта
* Перед використанням команд `msg`, `file`, `list` необхідно виконати `login`
* Ім'я отримувача повинно бути в списку активних користувачів
* Якщо користувач неактивний — повідомлення не буде доставлено

---

## 📖 Приклад використання

```text
login user1 123
list
msg user2 Привіт!
file user2 C:\test.txt
```
